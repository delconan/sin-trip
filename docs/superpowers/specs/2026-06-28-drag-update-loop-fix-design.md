# 拖拽递归更新修复设计

## 问题

拖动行程卡片时，`DropIndicator` 以 58px 高度插入卡片列表。插入提示会改变可放置卡片的矩形位置，使 dnd-kit 的 `overId` 在目标卡片和日期容器之间来回切换。每次切换都会触发 `onDragOver` 写入新的 `dropProjection`，再次改变布局，最终触发 React 的 `Maximum update depth exceeded`。

报错落在 `ScheduledCard` 渲染位置只是循环重渲染的结果，不是卡片组件主动递归更新。

## 修复设计

- 保留拖拽浮层和“放在这里”提示。
- 把插入提示改为零布局高度的悬浮标记，不再撑开卡片列表。
- 提示线与标签通过绝对定位绘制在目标卡片边界，不参与 dnd-kit 的碰撞几何计算。
- 拖拽投影、上下半区判定和最终 `move` 提交逻辑保持不变。
- 不增加节流、定时器或目标锁定状态，避免掩盖由布局反馈造成的根因。

## 测试

- 组件测试验证插入提示使用不改变布局的专用结构。
- 端到端测试在跨天拖拽期间收集 `pageerror`，确认没有 `Maximum update depth exceeded`。
- 保留现有跨天精准插入断言，确保修复没有牺牲放置位置。
- 运行单元测试、类型检查、Lint、生产构建和 Playwright。

## 验收标准

- 跨天或同天拖拽不再触发 React 最大更新深度错误。
- 拖动时仍能看见明确的插入位置提示。
- 放开后卡片落在提示对应的位置。
