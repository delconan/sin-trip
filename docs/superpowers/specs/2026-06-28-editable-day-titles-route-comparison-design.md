# 可编辑每日标题与交通方案对比设计

## 目标

在现有四日拖拽行程看板中增加两项能力：每日标题可原位编辑并跨设备保存；点击任一交通条时，不再直接跳转 Google Maps，而是打开步行、公交/MRT、打车三种方案的时间、距离与 2 大 2 小家庭预估总价对比。

## 每日标题编辑

- 默认沿用当前四个底稿标题。
- 点击标题后原位切换为单行输入框并自动聚焦。
- `Enter` 或失焦保存，`Escape` 取消；去除首尾空格，空标题不保存，最长 40 个字符。
- 保存后增加 `trip.revision`，沿用现有本地存储、原子版本冲突与 Realtime 同步流程。
- `TripState` 新增按 ISO 日期索引的 `dayTitles`。Supabase 的 `trips` 表新增 `day_titles jsonb`，RPC 在同一事务内写入标题、卡片和行程项目。
- 旧的本地数据或尚未升级的云端数据缺少 `dayTitles` 时，读取层以底稿标题补齐，避免破坏已有行程。

## 交通方案对比

- 现有交通条由外链改为按钮，仍显示系统推荐方案和时长。
- 点击后打开响应式对比弹层：桌面为居中纸张式对话框，手机为底部抽屉。
- 标题显示起点、终点和出发时间；主体固定展示步行、公交/MRT、打车三张方案卡。
- 每张卡显示图标、时长、距离、家庭预估总价、简短说明和“Google Maps 查询”按钮。
- 推荐方案使用当前规则：短距离优先步行；夜间、携带行李或公交比打车慢超过 20 分钟时优先打车；其他情况优先公交。
- 弹层支持关闭按钮、遮罩点击和 `Escape`，使用 `role="dialog"`、可读标题及键盘焦点样式。
- OneMap 成功时使用其时长和距离；失败时继续使用现有本地估时。Google Maps 仅在用户明确点击某方案按钮后打开。

## 家庭价格估算

- 步行：`S$0`。
- 公交/MRT：按 Public Transport Council 2026 票价表和路线距离计算。下限为 2 位成人普通卡价加 2 位儿童优惠价，上限为 4 位普通成人价；界面注明儿童优惠需符合资格并持有效优惠卡。
- 打车：以 ComfortDelGro 普通四座出租车当前起步价、距离跳表价为基础，按路线距离计算基础表价；用区间覆盖候车、路况和常见时段附加费。ERP、机场费、预约费和动态一口价不纳入，界面明确提示以叫车应用最终报价为准。
- 金额统一四舍五入到 S$0.10，显示为家庭总价区间；价格来源与核对日期 `2026-06-28` 在弹层底部展示。
- 费用计算为纯函数，便于对不同距离、儿童资格上下限和出租车附加费进行单元测试。

## 组件与数据边界

- `EditableDayTitle`：只负责编辑状态、校验和提交标题。
- `RouteComparisonDialog`：接收起终点、出发时间和三个 `RouteOption`，负责对比展示，不负责请求。
- `RouteRibbon`：继续请求/缓存路线，管理弹层开关，并把三个方案传给弹层。
- `RouteOption` 增加家庭价格估算字段，或通过独立的 `estimateFamilyRouteFare(route, context)` 在展示前派生，避免把临时费用写入路线缓存。
- `tripReducer` 新增 `set-day-title` action；状态补齐逻辑集中在 hydration 边界。

## 错误与降级

- OneMap 请求失败时，对比弹层仍展示三种本地估算方案。
- 缺少某种 OneMap 路线时，用本地估算补齐，保证始终有三张卡。
- 云端标题保存发生版本冲突时，沿用现有最新版回填机制，不静默覆盖其他设备。
- 费用始终标为“预估”，不把本地算法表达成实时 Grab、CDG Zig 或 Google 报价。

## 测试与验收

- 单元测试：标题校验与 reducer、旧状态补齐、公交家庭价上下限、出租车距离价与区间。
- 组件测试：点击标题编辑、Enter 保存、Escape 取消、空值拒绝；交通条打开弹层、三种方案存在、价格和 Google 按钮正确。
- 集成检查：API 返回 OneMap 或本地估算时均补齐三种方案；标题经 state API 和 RPC 持久化。
- 端到端测试：桌面和手机均可打开/关闭交通弹层；点击交通条本身不会导航，只有 Google Maps 按钮会产生外链。
